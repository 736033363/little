【目标】 让自己具备定制css框架的能力 【css组织】 base.css+common.css+page.css base.css 与UI无关，最底层 common.css 网站模块 page.css 页面相关，通用性较差 【css-library.css 大纲】 【reset.css】 提供一个更合适的平台，例如避免浏览器不一致、扫清一些障碍等等 【grid.css】 需求 1. 12列，与bootstrap相同 2. 分4种屏幕大小 <668 xs （手机，iPhone6/7/8是375*667） >=668 sm （平板） >=960 md （PC） >=1024 lg（大屏） 3. 添加通用屏。例如我的需求是一行4列，但在手机下是1行1列，我就可以这么写：class="col_4 xs_col_1"，通用屏比4种屏幕优先级更低。这里与bootstrap不同，bs只有四种，我这里其实是5种，我认为我的更清晰 4. 栅格只做容器，不支持作为功能类，这样更简单 实现需要注意的点： 必须包含在.g-grid中，width需额外定义 行.row必须放在.g-grid中，默认抵消.g-grid内边距，默认行与行间距20px 内容应该放在列中，列可以放在行中，也可以放在.g-grid中，列中也能放行，列与列有间距 【unit.css】 元件，通常是不能在分割的个体，例如按钮 【module.css】 模块，例如登录模块、导航 【skin.css】 通常是文字颜色、背景色 【function.css】 把某些常用的样式抽取出来
