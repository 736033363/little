离线存储

想来不难，但若需要做一个完整的产品，还需要深入研究

文件系统

类似java的文件操作，接口初看有些多且复杂；文件系统提供了一个永久存储空间，需要用户通过硬盘或应用删除，可以做离线视频等新功能；以后做相关产品可详细研究。

通信

跨文档消息传输，可以实现不同域的网页相互通信；WebSockets 提供了一个双向的，http不具备的功能，前端很简单，关键是服务端，应用场景是聊天室，以后可以研究；Server-Sent Events顾名思义，服务器端向客户端发送，例如实时股票在线数据，若有该需求，需要研究；

WebRTC

WebRTC，就是Web世界的实时通信。它的目标是浏览器与浏览器的实时通信，例如语音或视频聊天。getUserMedia访问本地设备（摄像头和麦克风）。这个功能点好像挺复杂，例如搭建信令、穿越防火墙等，很多东西都看不太懂，若以后需要做类似产品，需要好好研究。

扩展XMLHttpRequest API

接收二进制，例如保存一张图片到本地数据库；发送表单、发送Blog对象；跨域请求，在后端设置；

多线程 Web Workers

多线程就是前端的后台，用于有大数据需求且不影响用户操作；SharedWorker多个页面共享一个后台线程，不太懂里面的原理；若有需求需要深入研究下；

地理位置信息

只要浏览器和设备支持，就可以获取自己的位置，主要通过Geolocation；可以将本地位置通过Google地图显示出来；若碰到此类需求仍需仔细研究；

拖放API和通知API

拖放的核心就是我之前做的编辑器，若深入需要继续研究；通知需要获得用户权限，好像也需要是https；
